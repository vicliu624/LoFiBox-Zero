# LoFiBox Copyright And Resource Governance Specification

## 1. Purpose

This document defines copyright, licensing, and resource-origin rules for `LoFiBox Zero`.
Official archive readiness requires copyright clarity for code, assets, fixtures, screenshots, documentation, and third-party material.

## 2. Source Licensing

- Source files should carry SPDX license headers.
- `LICENSE` or `COPYING` must clearly state the project license.
- `debian/copyright` must use machine-readable copyright format.
- Project-owned source, scripts, documentation, man pages, CI files, and CMake files must use `SPDX-License-Identifier: GPL-3.0-or-later`.
- Desktop, AppStream, and MIME metadata files must use `SPDX-License-Identifier: CC0-1.0`.
- Project-created installable logo and UI artwork must use `SPDX-License-Identifier: GPL-3.0-or-later` unless a future asset-specific license decision is recorded.
- Third-party source files may keep upstream headers unchanged, but must be registered in `docs/third-party.md` and `debian/copyright`.
- `scripts/check-spdx.ps1` is the local and CI gate for project-owned SPDX coverage.

## 3. Asset Licensing

The project must track license and origin for:

- icons
- logos
- themes
- fonts
- test audio
- screenshots
- documentation
- sample data

Unknown-license assets are blockers for official archive readiness.

## 4. Forbidden Resource Use

The project must not use:

- copyright-unclear commercial songs
- copyright-unclear album covers
- copyright-unclear fonts
- copyright-unclear icons
- AppStream screenshots containing unlicensed commercial music information or album artwork
- precompiled binary blobs without explicit governance and license clearance

## 5. Test Fixtures

Test audio should be:

- generated by the project
- or taken from clearly free-licensed sources with documented origin and license

Fixture metadata and artwork must be governed like code and assets.

## 6. Third-Party Register

Third-party code and resources must be documented in:

- `docs/third-party.md`
- `debian/copyright`

The documentation must include source, license, and reason for inclusion.

## 7. Font Policy

LoFiBox should use system font packages for CJK text rendering.

- CJK font binaries must not be vendored merely for convenience.
- Debian packaging should prefer `fonts-noto-cjk` or another distribution-packaged CJK font.
- Runtime code may probe common system font paths or use a platform font discovery layer.
- Reintroducing a vendored font binary requires an explicit source, version, license, and copyright review.
