# LoFiBox GitHub Pages APT Repository Specification

## 1. Purpose

This document defines the temporary LoFiBox-maintained APT repository used before
LoFiBox is accepted into the official Debian archive.

The long-term target remains the official Debian archive. The GitHub Pages APT
repository is a preview distribution channel for early users and device testing.
It must not weaken Debian archive readiness.

## 2. Boundary

The preview repository is:

- a standard signed APT repository
- hosted as static files through GitHub Pages
- generated from Debian package artifacts
- consumed through a per-repository `Signed-By` keyring

The preview repository is not:

- a replacement for official Debian packaging
- a hand-written package index
- a `.deb` download page
- a reason to vendor dependencies
- a reason to use build-time network fetches
- a place to store signing private keys in git

## 3. Repository Shape

The GitHub Pages artifact must contain:

```text
public/
  .nojekyll
  lofibox-archive-keyring.pgp
  debian/
    dists/<suite>/InRelease
    dists/<suite>/Release
    dists/<suite>/Release.gpg
    dists/<suite>/main/binary-<arch>/
    pool/
```

The user-facing APT URI for the current repository is:

```text
https://vicliu624.github.io/lofibox-apt/debian
```

A future custom domain such as `https://apt.lofibox.app/debian` may replace the
GitHub Pages project URL without changing the repository format.

## 4. Signing Rules

The repository must publish signed metadata:

- `InRelease`
- `Release`
- `Release.gpg`

The signing private key must be stored outside git. GitHub Actions may use it
only through protected repository secrets:

- `LOFIBOX_APT_GPG_PRIVATE_KEY`
- `LOFIBOX_APT_GPG_KEY_ID`

The public key must be exported as:

```text
lofibox-archive-keyring.pgp
```

User documentation must use deb822 `.sources` files with `Signed-By`.
Documentation must not use `apt-key`.

## 5. Version Policy

Preview repository package versions must leave room for future Debian official
archive packages.

Use local preview suffixes:

```text
0.1.0-1~lofibox1
0.1.0-1~lofibox2
0.1.1-1~lofibox1
```

This lets a future official Debian package such as `0.1.0-1` naturally supersede
`0.1.0-1~lofibox1`.

## 6. Suite And Architecture Policy

Publish suites by explicit codename:

```text
trixie
bookworm
forky
```

Do not publish an ambiguous `stable` suite.

The repository must support multiple architectures as package artifacts become
available. The current preview repository must publish `amd64`, `arm64`, and
`armhf` package indexes when those package artifacts are available.

The architecture meanings are:

- `amd64`: desktop and x86_64 Linux validation hosts
- `arm64`: Raspberry Pi Compute Module 4/5 class 64-bit systems
- `armhf`: Raspberry Pi Compute Module 0 / ARMv6 hard-float systems

Cross-built package artifacts must keep build-machine tools and target libraries
separate. Build tools such as `pkgconf` belong to the build architecture and must
be declared with a native build-dependency qualifier when needed. Target
development libraries such as `libx11-dev`, `libxkbcommon-dev`, and
`libfreetype-dev` belong to the host architecture selected by `--host-arch`.

The script-level repository generator accepts repeated `.changes` files so
`amd64`, `arm64`, and `armhf` artifacts can be included in the same published
snapshot.

## 7. Tooling

Repository metadata must be generated by repository tooling such as `aptly`.
Do not hand-write `Packages`, `Release`, `InRelease`, or checksums.

The project-provided generator is:

```text
scripts/build-github-pages-apt-repository.sh
```

The GitHub Actions publisher is:

```text
lofibox-apt/.github/workflows/publish.yml
```

## 8. CI And Publication

Normal CI must continue to validate Debian package buildability, install
skeletons, desktop metadata, lintian, and autopkgtest.

The preview APT publication workflow is a manual release workflow. It must not
run on pull requests and must not expose signing secrets to untrusted code.

## 9. Migration To Official Debian

When LoFiBox becomes available from the official Debian archive, the preview
repository should become a preview/nightly channel. Stable user instructions
should direct users back to the official Debian repository.
